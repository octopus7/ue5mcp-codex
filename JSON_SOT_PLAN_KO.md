# JSON SoT 향후 계획

## 상태

연기됨. 초기 버전은 `patch` 모드를 기본이자 유일한 필수 워크플로로 사용합니다.

## 연기한 이유

- `patch` 모드는 Widget Editor에서 수행한 수동 수정을 더 잘 보존합니다.
- 전체 덮어쓰기 의미론을 피하면 초기 릴리스 리스크가 더 낮습니다.
- 더 안전한 기본값 덕분에 팀 도입이 더 쉽습니다.

## 계획된 SoT 모델(추후)

- JSON에서 `mode: "sot"`를 명시적으로 선언
- SoT 실행은 드라이런 미리보기와 명시적 덮어쓰기 확인이 필수
- 적용 전 충돌 및 드리프트 리포팅이 필수
- 파괴적 변경을 제한하기 위한 안전한 스코프 규칙 정의

## 구현 전 선행 조건

- 성숙한 패치 워크플로 및 복구 신뢰성
- 위젯 트리/바인딩 작업에 대한 충분한 회귀 테스트 커버리지
- 덮어쓰기 및 롤백 정책에 대한 팀 승인
